# **Лабораторная №6 — Дизайн-система и применение в приложении**

### **Задача**

Сделать мини дизайн-систему для приложения и применить её в экранах, которые уже есть в проекте (авторизация + список и/или экран деталей/заглушка фичи).

Цель лабы — чтобы UI выглядел единообразно и изменения (цвет/шрифт/отступы/стиль кнопок) вносились централизованно, а не “по месту”.

---

## **Требования (минимум)**

### **1) Токены дизайн-системы**

Нужно выделить и оформить в коде набор токенов (минимум):

- **Colors** (например background, primary, secondary, textPrimary, textSecondary, error)
    
- **Typography** (стили текста: title, body, caption, и т.п.)
    
- **Spacing** (отступы/радиусы: s, m, l или 4/8/16/24)
    
- (по желанию) **Shadows / CornerRadius** как отдельные константы
    

Пример (можно иначе):

```swift
enum DS {
    enum Colors {
        static let background = UIColor.systemBackground
        static let primary = UIColor.systemBlue
        static let textPrimary = UIColor.label
        static let error = UIColor.systemRed
    }

    enum Spacing {
        static let s: CGFloat = 8
        static let m: CGFloat = 16
        static let l: CGFloat = 24
        static let cornerRadius: CGFloat = 12
    }

    enum Typography {
        static func title() -> UIFont { .systemFont(ofSize: 20, weight: .semibold) }
        static func body() -> UIFont { .systemFont(ofSize: 16, weight: .regular) }
        static func caption() -> UIFont { .systemFont(ofSize: 13, weight: .regular) }
    }
}
```

### **2) Компоненты (минимум 3)**


Нужно сделать минимум **3 переиспользуемых UI-компонента**, которые используют токены и конфигурируются данными.

Можно использовать любую публичную дизайн систему, оформленную по всем канонам (например, Apple HIG в Figma или еще что-то), можно оформить свои компоненты без публичного дизайна, решать только вам.


Подходящие варианты (можно выбрать любые 3):

- DSButton (primary/secondary/disabled)
    
- DSTextField (с заголовком/плейсхолдером/ошибкой)
    
- DSErrorView (текст + retry-кнопка)
    
- DSLoadingView (индикатор + текст)
    
- DSEmptyView
    
- InfoRowView (пара “лево/право”)
    
- SectionHeaderView
    

P.S: Нейминг примерный, можно выбирать любой другой

Пример (не копировать бездумно):

```swift
final class DSButton: UIButton {
    enum Style { case primary, secondary }

    private let style: Style

    init(style: Style) {
        self.style = style
        super.init(frame: .zero)
        applyStyle()
    }

    required init?(coder: NSCoder) { fatalError() }

    private func applyStyle() {
        layer.cornerRadius = DS.Spacing.cornerRadius
        titleLabel?.font = DS.Typography.body()

        switch style {
        case .primary:
            backgroundColor = DS.Colors.primary
            setTitleColor(.white, for: .normal)
        case .secondary:
            backgroundColor = .clear
            setTitleColor(DS.Colors.primary, for: .normal)
        }
    }
}
```

### **3) Применение на экранах**

Нужно применить дизайн-систему минимум на **двух** экранах, например:

- экран авторизации (Лаба 3)
    
- экран списка (Лаба 5)
    

Требования:

- убрать “ручные” стили (случайные цвета/шрифты/радиусы) и заменить на DS
    
- экраны должны выглядеть единообразно
    

### **4) Состояния экрана через компоненты**


Для списка и/или деталей нужно использовать DS-компоненты для состояний:

- loading
    
- error (+ retry)
    
- empty (если уместно)
    


Можно сделать отдельные view (DSLoadingView, DSErrorView, DSEmptyView) или один контейнер.

### **5) Архитектура**

- Дизайн-система должна быть вынесена отдельно (папка/модуль DesignSystem).
    
- UI-компоненты не должны содержать бизнес-логику.
